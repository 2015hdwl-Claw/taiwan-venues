## 1. 目標與範圍

### 目標
確保台灣會議場地資料庫的**完整性**與**準確確性**。

### 核心原則
> **資料對了才有價值，資料錯了一文不值。**

### 驗證範圍
- 全台 22 縣市
- 總場地數：490 個
- 驗證項目：官網、會議室頁面、照片、狀態

---

## 2. 驗證流程

### Phase 1: 官網驗證
```
1. 開啟官網 URL
2. 檢查 HTTP 狀態碼
3. 檢查網頁標題是否匹配
4. 記錄錯誤類型（DNS、SSL、Timeout 等）
```

### Phase 2: 會議室頁面尋找
```
1. 搜尋關鍵字：會議、宴會、場地、租借、活動
2. 找到會議室相關連結
3. 開啟會議室頁面
4. 記錄 meetingPageUrl
```

### Phase 3: 照片抓取
```
1. 在會議室頁面抓取照片
2. 過濾條件：
   - 寬度 > 200px
   - 高度 > 150px
   - 排除 logo、icon、svg
3. 記錄照片 URL（最多 10 張）
```

### Phase 4: 狀態判定
```
1. 根據驗證結果判定狀態
2. 更新資料庫欄位
3. 標記問題類型
```

## 3. 狀態定義

### status 欄位（後台分類用）

| 狀態 | 值 | 說明 | 後台顯示 |
|------|-----|------|---------|
| **上架** | `上架` | 營運中，可租借 | ✅ 綠色 |
| **下架** | `下架` | 已歇業或停止營運 | ❌ 紅色 |
| **待修** | `待修` | 資料有誤需修正 | ⚠️ 黃色 |
| **不提供會議** | `不提供會議` | 非會議場地（如咖啡廳） | 🚫 灰色 |

### verificationStatus 欄位（驗證細節）

| 狀態 | 說明 |
|------|------|
| `已驗證` | 官網、會議室頁面、照片都正確 |
| `待手動驗證` | 被防護系統阻擋（如 Cloudflare） |
| `待確認` | 官網無法連線，需確認營運狀態 |
| `待修正` | 會議室頁面或照片有問題 |

## 4. 驗證標準

### ✅ 驗證成功（OK）
- [x] 官網可開啟
- [x] 找到會議室頁面
- [x] 抓取到會議室照片
- [x] 標題匹配或合理

### ⚠️ 部分成功（部分OK）
- [x] 官網可開啟
- [x] 找到會議室頁面
- [ ] 缺少照片

### ❌ 驗證失敗（有問題）
- [x] 官網可開啟
- [ ] 找不到會議室頁面
- [ ] 可能不提供會議服務

### ❌ 驗證錯誤（錯誤）
- [ ] 官網無法開啟
- [ ] DNS 錯誤 / SSL 錯誤 / Timeout

## 5. 問題分類

### 5.1 DNS 無法解析
- **原因**: 官網域名已失效
- **處理**: 
  - 搜尋新官網
  - 確認是否歇業
  - 標記為「待確認」或「下架」

### 5.2 官網標題不匹配
- **原因**: 可能指向錯誤官網或集團首頁
- **處理**: 
  - 手動確認官網正確性
  - 更新為正確的會議室頁面 URL

### 5.3 找不到會議室頁面
- **原因**: 
  - 網站無會議室資訊
  - 純餐飲或展館
  - 需登入才能查看
- **處理**: 
  - 確認是否提供會議服務
  - 標記為「待修」或「不提供會議」

### 5.4 缺少會議室照片
- **原因**: 
  - 會議室頁面無照片
  - 照片載入失敗
- **處理**: 
  - 手動補充照片
  - 或標記為「待修」

### 5.5 SSL 憑證錯誤
- **原因**: 憑證過期或設定錯誤
- **處理**: 
  - 嘗試使用 HTTP
  - 通知業者

### 5.6 連線逾時
- **原因**: 網站回應過慢
- **處理**: 
  - 增加逾時時間重試
  - 手動驗證

### 5.7 防護系統阻擋
- **原因**: Cloudflare、Akamai 等
- **處理**: 
  - 標記為「待手動驗證」
  - 這些場地確定有會議室

## 6. 批次驗證

### 6.1 批次規劃

| 批次 | 縣市 | 預計場地數 | 狀態 |
|------|------|-----------|------|

| 2 | 新北市 | ~30 | ⏳ 待驗證 |
| 3 | 台中市 | ~40 | ✅ 已完成 |
| 4 | 高雄市 | ~35 | ✅ 已完成 |
| 5 | 台南市 | ~25 | ⏳ 待驗證 |
| 6 | 桃園市 | ~20 | ⏳ 待驗證 |
| 7 | 其他縣市 | ~100 | ⏳ 待驗證 |

### 6.2 每批次驗證步驟

```bash
# 1. 產生待驗證清單
node generate-verify-list.js <縣市>

# 2. 執行驗證
node verify-sop-v3.js <清單檔案>

# 3. 檢查結果
# - 產生驗證報告
# - 更新資料庫

# 4. 提交複審
# - 推送到 GitHub
# - 通知用戶複審
```

### 6.3 批次完成標準

每批次完成後需確認：
- [ ] 所有場地已驗證
- [ ] 狀態已更新（上架/下架/待修）
- [ ] 驗證結果已記錄
- [ ] 已推送到 GitHub
- [ ] 已通知用戶複審

```json

  "status": "上架 | 下架 | 待修 | 不提供會議",
  "lastVerified": "2026-03-02T00:00:00.000Z",
  "verifiedTitle": "網頁標題",
  "meetingPageUrl": "會議室頁面 URL",
  "images": {
    "main": "主照片 URL",
    "gallery": ["照片1", "照片2", "..."],
    "source": "照片來源頁面",
    "verified": true,
    "verifiedAt": "2026-03-02T00:00:00.000Z"

  "verificationNote": "驗證備註",
```

### 7.2 狀態更新規則

| 驗證結果 | status | verificationStatus |
|---------|--------|-------------------|

| 部分OK（無照片） | 上架 | 已驗證 |
| 找不到會議室頁面 | 待修 | 待修正 |
| DNS 錯誤 | 待修 | 待確認 |
| 防護阻擋 | 上架 | 待手動驗證 |

## 8. 後台分類

### 8.1 後台網址
https://2015hdwl-claw.github.io/taiwan-venues/admin.html

### 8.2 分類邏輯

```javascript
// 後台分類依據 status 欄位
function getCategory(venue) {
  switch (venue.status) {
    case '上架':
      return 'active';      // 綠色，可租借
    case '下架':
      return 'closed';      // 紅色，已歇業
    case '待修':
      return 'needsFix';    // 黃色，需修正
    case '不提供會議':
      return 'notMeeting';  // 灰色，非會議場地
    default:
      return 'unknown';     // 未分類
  }
}
```

## 9. 特殊情況處理

### 9.1 國際連鎖酒店（被防護阻擋）
- 萬豪集團（W飯店、艾美酒店）
- 洲際集團（晶華酒店、英迪格酒店）
- 希爾頓集團（寒舍艾麗）
- 凱悅集團（君悅酒店）

**處理方式**:
- 標記為「上架」
- verificationStatus = 「待手動驗證」
- 手動補充會議室頁面 URL

### 9.2 已歇業場地
- 確認歇業後標記為「下架」
- 記錄歇業時間（如已知）

### 9.3 咖啡廳等非會議場地
- 標記為「不提供會議」
- 從會議場地清單排除

### 9.4 重複場地
- 保留最完整的紀錄
- 刪除重複項
