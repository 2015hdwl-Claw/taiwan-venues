# 台灣會議場地資料庫 - Total Solution

## 🎯 專案目標

建立一個**完整、準確、持續更新**的台灣會議場地資料庫，提供 total solution 給使用者。

---

## 📋 Total Solution 包含

### 1. 資料收集（Data Collection）

#### 1.1 場地來源
- **官方網站**：各場地的官方網站（最準確）
- **維基百科**：知名場地的公開資訊（備援）
- **政府資料**：機關場地的公開資料
- **人工審核**：重要場地的二次確認

#### 1.2 資料類型
- **基本資訊**：名稱、地址、聯絡方式
- **場地資訊**：會議室名稱、容納人數、設備
- **價格資訊**：時薪、半日價、全日價
- **照片資訊**：主照片、會議室照片、平面圖
- **營運資訊**：開放時間、預約方式、狀態

#### 1.3 資料結構
```json
{
  "id": 1001,
  "name": "場地名稱(英文簡稱)",
  "roomName": "會議室名稱",
  "venueType": "場地類型",
  "city": "城市",
  "address": "完整地址",
  "contactPerson": "聯絡人",
  "contactPhone": "電話",
  "contactEmail": "Email",
  "url": "官方網站",
  "pricePerHour": 時薪,
  "priceHalfDay": 半日價,
  "priceFullDay": 全日價,
  "maxCapacityTheater": 劇院式容納,
  "maxCapacityClassroom": 教室式容納,
  "availableTimeWeekday": "平日時間",
  "availableTimeWeekend": "假日時間",
  "equipment": "設備",
  "images": {
    "main": "主照片網址",
    "gallery": ["會議室照片1", "會議室照片2"],
    "floorPlan": "平面圖網址",
    "source": "照片來源"
  },
  "status": "上架/下架",
  "verified": true,
  "verifiedAt": "驗證日期"
}
```

---

### 2. 資料驗證（Data Validation）

#### 2.1 網址驗證
- **第1層**：HTTP 狀態檢查 + 關鍵字匹配
- **第2層**：搜尋引擎驗證（Google/Bing）
- **第3層**：人工審核（重要場地）

#### 2.2 資料完整性檢查
- 必填欄位：名稱、地址、電話
- 建議欄位：Email、官網、照片
- 品質檢查：價格合理性、容納人數合理性

#### 2.3 照片驗證
- 真實性：照片網址必須可訪問
- 相關性：照片必須與場地相關
- 數量：每個場地至少2張照片

---

### 3. 資料爬蟲（Data Crawling）

#### 3.1 智能爬蟲策略
```
判斷網頁類型
  ├─ 靜態網頁 → web_fetch（快速、省資源）
  └─ 動態網頁 → agent-browser（處理 JS 渲染）
```

#### 3.2 爬蟲工具
- **web_fetch**：靜態 HTML 提取（適合 30-40% 網站）
- **agent-browser**：JavaScript 渲染處理（適合 20-30% 網站）
- **Wikipedia API**：公開資訊備援
- **手動補充**：無法自動化的場地

#### 3.3 爬蟲流程
```
1. 驗證網址正確性
2. 嘗試 web_fetch（靜態）
   ├─ 成功 → 提取照片
   └─ 失敗 → 啟動 agent-browser
3. 提取場地資訊
4. 驗證照片品質
5. 儲存到資料庫
```

---

### 4. 資料管理（Data Management）

#### 4.1 版本控制
- **Git 儲存**：所有變更都有記錄
- **Commit 訊息**：清楚的變更說明
- **定期備份**：避免資料遺失

#### 4.2 更新流程
```
1. 發現資料錯誤或缺失
2. 驗證修正資訊
3. 更新 JSON 檔案
4. Commit + Push
5. 自動部署到 GitHub Pages
```

#### 4.3 品質監控
- **每日檢查**：admin.html 查看資料狀態
- **每週審核**：檢查重要場地的準確性
- **每月更新**：批量爬蟲更新照片

---

### 5. 資料展示（Data Presentation）

#### 5.1 後台管理（admin.html）
- 場地列表（支援搜尋、篩選）
- 詳細資訊展示
- 照片預覽 + 網址顯示
- 問題標記（缺電話、缺照片等）

#### 5.2 API 介面
- **GitHub Raw JSON**：`venues-all-cities.json`
- **免費託管**：GitHub Pages
- **自動更新**：Push 後立即生效

---

### 6. 持續維護（Continuous Maintenance）

#### 6.1 自動化任務
- **網址驗證**：每月執行一次
- **照片檢查**：每月執行一次
- **資料備份**：每週執行一次

#### 6.2 人工審核
- **重要場地**：Top 50 場地每季審核
- **新增場地**：新場地的人工驗證
- **使用者回報**：處理錯誤回報

#### 6.3 擴充計畫
- **更多場地**：從 440 筆擴充到 1000+ 筆
- **更多資訊**：交通資訊、周邊設施
- **更多功能**：線上預約、即時查詢

## 📝 維護指南

### 如何新增場地
1. 收集場地基本資訊
2. 驗證官網網址
3. 爬取照片和詳細資訊
4. 更新 `venues-all-cities.json`
5. Commit + Push

### 如何修正錯誤
1. 在 `venues-all-cities.json` 找到場地
2. 修正錯誤資訊
3. 驗證修正結果
4. Commit + Push

### 如何更新照片
1. 使用 `smart_crawl_photos.py` 爬取
2. 或手動查找照片網址
3. 更新 `images` 欄位
4. Commit + Push
