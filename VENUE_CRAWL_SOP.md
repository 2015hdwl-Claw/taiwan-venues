# 場地網站爬蟲標準程序 (SOP)

**建立日期**：2026-03-01
**目的**：確保每次爬蟲都遵循正確步驟，不鬼打牆

---

## 🎯 目標資料（每個場地必收集）

### 1. 官網資訊
- [ ] 官網首頁網址
- [ ] 會議/宴會場地頁面網址

### 2. 聯絡資訊
- [ ] 聯絡電話
- [ ] 聯絡 Email
- [ ] 聯絡地址
- [ ] 業務聯絡窗口（如有）

### 3. 租借資訊
- [ ] 價格（時薪/半日/全日）
- [ ] 可租借時段（平日/假日）
- [ ] 容納人數（劇院式/教室式）
- [ ] 設備清單

### 4. 照片（最重要！）
- [ ] **主照片**：場地外觀（用於卡片顯示）+ 網址
- [ ] **會議室照片**：每個會議室的照片 + 網址

---

## 📋 標準爬蟲流程

### Step 1：確認官網
```
1. 用 web_search 搜尋「[場地名稱] 官網」
2. 確認官網網址正確
3. 記錄官網首頁
```

### Step 2：找會議/宴會場地頁面
```
1. 用 web_fetch 抓取官網首頁（靜態）
2. 搜尋關鍵字：會議、宴會、場地、租借、活動
3. 找到會議場地頁面連結
4. 如果 web_fetch 失敗 → 使用 browser（動態）
```

### Step 3：提取聯絡資訊
```
1. 從會議場地頁面提取：
   - 電話
   - Email
   - 地址
   - 租借時段
2. 如果頁面沒有 → 回首頁找聯絡我們
```

### Step 4：提取照片
```
1. 主照片（場地外觀）：
   - 通常在首頁或關於我們
   - 用於卡片顯示
   
2. 會議室照片：
   - 從會議場地頁面提取
   - 記錄每張照片的完整網址

3. 照片網址必須是完整 URL（不是相對路徑）
```

### Step 5：驗證與記錄
```
1. 驗證所有網址可訪問
2. 驗證照片網址有效
3. 記錄到 venues-all-cities.json
4. 更新進度到 CRAWL_PROGRESS.md
```

## 🔧 工具選擇準則

### web_fetch（優先使用）
- **適用**：靜態 HTML 網頁
- **優點**：快速、省資源
- **判斷**：頁面不需要 JavaScript 渲染

### browser（備援使用）
- **適用**：動態 JavaScript 網頁
- **觸發條件**：
  1. web_fetch 抓不到內容
  2. 頁面需要點擊/滾動才能顯示
  3. 內容是 JavaScript 動態生成

## ⚠️ 常見問題與解決

### Q1: web_fetch 抓不到內容
**解決**：改用 browser

### Q2: 找不到會議場地頁面
**解決**：
1. 搜尋「[場地名稱] 會議廳」
2. 搜尋「[場地名稱] 宴會廳」
3. 直接打電話詢問

### Q3: 照片是相對路徑
**解決**：組合完整 URL
- 相對路徑：`/images/photo.jpg`
- 完整 URL：`https://example.com/images/photo.jpg`

### Q4: 網站有反爬蟲機制
**解決**：
1. 使用 browser 模擬真人
2. 降低爬蟲速度
3. 手動補充資料
