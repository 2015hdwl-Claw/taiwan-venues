## 🔧 執行方式建議

### 自動驗證（web_fetch）- 55 個

**適合**:
- DNS 無法解析（嘗試搜尋新官網）
- SSL 憑證錯誤（改用 HTTP）

**優點**:
- 速度快（每個 2-3 分鐘）
- 可批量處理
- 自動記錄結果

**執行方式**:
```bash
# 使用 web_fetch 驗證
web_fetch <URL>
```

### 手動驗證（Chrome Extension）- 41 個

**適合**:
- 連線逾時（需要等待載入）
- JavaScript 錯誤（需要完整瀏覽器環境）

**優點**:
- 成功率高
- 可處理複雜網站
- 可截圖保存

**缺點**:
- 速度慢（每個 3-5 分鐘）
- 需要人工監控

**執行方式**:
1. 開啟 Chrome Extension
2. 輸入場地官網 URL
3. 等待載入（最多 30 秒）
4. 截圖保存
5. 記錄結果

### 搜尋確認 - 42 個

**適合**:
- DNS 無法解析且找不到官網
- 可能已歇業的場地

**方法**:
1. Google 搜尋場地名稱
2. 查看是否仍營運
3. 標記為「下架」或更新官網
