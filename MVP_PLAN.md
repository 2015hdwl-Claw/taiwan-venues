## 🎯 Day 2 計畫 (2026-03-10)

### 目標
建立 AI 對話系統，讓使用者可以用自然語言搜尋場地

### 任務清單
1. ⬜ 整合 GLM-4.7-Flash API
2. ⬜ 設計場地推薦 prompt
3. ⬜ 實作對話記憶（session-based）
4. ⬜ 整合搜尋 API 到 AI 回覆
5. ⬜ 測試對話流程

### 技術選擇
- **LLM**: GLM-4.7-Flash（快速、便宜）
- **備用**: DeepSeek-V3.2
- **對話記憶**: In-memory session store

### Prompt 設計草案
```
你是台灣場地搜尋助手。使用者會描述他們的需求，你需要：

1. 理解使用者需求（人數、預算、地點、設備）
2. 呼叫搜尋 API
3. 推薦最適合的 3-5 個場地
4. 提供場地資訊（名稱、地點、價格、容納人數）
5. 詢問是否需要更多資訊或預訂

回覆要：
- 簡潔明瞭
- 重點突出
- 友善專業
```

## 🎯 Day 3 計畫 (2026-03-11)

### 目標
建立用戶介面（LINE Bot 或 Web UI）

### 選項 A: LINE Bot
- **優點**: 用戶熟悉、易於分享
- **缺點**: 審核時間長（可能 3-7 天）
- **風險**: 高

### 選項 B: Web UI
- **優點**: 快速上線、無需審核
- **缺點**: 需要推廣
- **風險**: 低

### 決策
**優先開發 Web UI**，同時申請 LINE Bot

### 任務清單
1. ⬜ 建立簡易 Web UI (HTML + JS)
2. ⬜ 串接搜尋 API
3. ⬜ 串接 AI 對話 API
4. ⬜ 設計用戶流程
5. ⬜ 測試端到端
