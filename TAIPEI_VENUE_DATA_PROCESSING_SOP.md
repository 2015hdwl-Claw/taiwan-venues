## 2. 資料來源識別

### 2.1 優先順序

| 順序 | 來源類型 | 優先級 | 說明 |
|------|---------|--------|------|
| 1 | **官方網站** | ✅ 最高 | 最可靠的資料來源 |
| 2 | **官方 Facebook** | ✅ 高 | 可補充照片和最新資訊 |
| 3 | **官方 Instagram** | ✅ 高 | 可補充照片 |
| 4 | **Google Maps** | ⚠️ 中 | 僅在無其他來源時使用 |
| 5 | **Wikipedia** | ❌ 禁止 | 容易過時、不準確 |
| 6 | **新聞網站** | ❌ 禁止 | 容易過時 |

### 2.2 官網結構分析

#### A. 飯店場地（87 間）

**常見官網結構**：
```
首頁
├── 宴會/會議（通常在主選單）
│   ├── 會議室列表（我們要找的頁面）
│   │   ├── 會議室 1 詳細頁
│   │   ├── 會議室 2 詳細頁
│   │   └── ...
│   └── 宴會廳列表
└── 聯絡我們
```

**搜尋關鍵字**：
- 中文：會議、宴會、場地、租借、活動、場地租賃
- 英文：meeting, conference, venue, banquet, event, ballroom

**常見會議室頁面 URL 模式**：
- `/meeting/`
- `/conference/`
- `/venue/`
- `/banquet/`
- `/event-space/`
- `/meetings-events/`

#### B. 會議中心（90 間）

**常見官網結構**：
```
首頁
├── 場地介紹/會議室（主要頁面）
│   ├── 場地列表
│   ├── 價格方案
│   └── 設備說明
├── 預約/線上查詢
└── 聯絡我們
```

**搜尋關鍵字**：
- 中文：場地、會議室、教室、研習室、演講廳
- 英文：venue, room, classroom, auditorium

#### C. 展演場地（22 間）

**常見官網結構**：
```
首頁
├── 場地租借
│   ├── 場地介紹
│   ├── 租借辦法
│   └── 價格表
└── 檔期查詢
```

**搜尋關鍵字**：
- 中文：場地租借、展演空間、劇場、音樂廳
- 英文：rental, venue, space, theater, hall

## 3. 資料擷取流程

### 3.1 使用的工具

| 工具 | 用途 | 指令 |
|------|------|------|
| **Playwright** | 瀏覽器自動化 | `npx playwright` |
| **Python requests** | HTTP 請求 | `python3 verify_and_update_urls.py` |
| **web_fetch** | 快速網頁抓取 | OpenClaw 內建工具 |
| **browser** | 互動式驗證 | OpenClaw 內建工具 |

### 3.2 擷取欄位清單

#### 必要欄位（不可缺失）

```json
{
  "name": "場地名稱",
  "roomName": "會議室名稱",
  "venueType": "場地類型",
  "city": "台北市",
  "address": "完整地址",
  "contactPhone": "聯絡電話",
  "url": "官方網站 URL"
}
```

#### 重要欄位（盡量補充）

```json
{
  "venueListUrl": "會議室清單頁面 URL",
  "venueMainImageUrl": "場地主照片 URL",
  "images": {
    "main": "主照片 URL",
    "gallery": ["照片 1", "照片 2"],
    "source": "照片來源頁面 URL",
    "verified": true,
    "verifiedAt": "2026-03-05T00:00:00Z"
  },
  "priceHalfDay": 30000,
  "priceFullDay": 50000,
  "maxCapacityTheater": 100,
  "maxCapacityClassroom": 60,
  "roomsCount": 5,
  "roomNames": ["會議室 A", "會議室 B"]
}
```

### 3.3 不同網站結構的處理方式

#### 類型 A：單頁式會議室列表

**特徵**：所有會議室資訊在一個頁面

**擷取方式**：
1. 開啟會議室列表頁面
2. 抓取所有會議室名稱、容量、價格
3. 記錄 venueListUrl

**範例**：TICC 台北國際會議中心
```
URL: https://www.ticc.com.tw/wSite/sp?xdUrl=/wSite/ap/lp_VenueSearch.jsp
```

#### 類型 B：多頁式會議室詳情

**特徵**：每個會議室有獨立頁面

**擷取方式**：
1. 開啟會議室列表頁面
2. 抓取所有會議室連結
3. 逐一開啟每個會議室頁面
4. 抓取詳細資訊

**範例**：台北喜來登大飯店
```
列表頁: https://www.sheratongrandtaipei.com/meetings
詳細頁: https://www.sheratongrandtaipei.com/meetings/room-1
```

#### 類型 C：PDF 價目表

**特徵**：價格資訊在 PDF 檔案中

**擷取方式**：
1. 下載 PDF 檔案
2. 手動或自動解析 PDF
3. 整理成結構化資料

**範例**：部分政府場地

#### 類型 D：需要登入/查詢

**特徵**：需要選擇日期才能看到空缺

**擷取方式**：
1. 使用 Playwright 模擬操作
2. 或手動查詢後記錄

### 3.4 照片擷取規則

#### 照片來源優先順序

1. **會議室頁面的照片**（最準確）
2. **官網首頁的場地照片**
3. **Facebook 專頁照片**
4. **Google Maps 照片**（僅在無其他來源時）

#### 照片篩選條件

```python
# 過濾條件
- 尺寸 > 200x150（排除 icon、logo）
- 不包含關鍵字：logo, icon, avatar, sprite, banner
- 來源必須是官網或官方社群
```

#### 照片記錄格式

```json
{
  "venueMainImageUrl": "https://example.com/main.jpg",
  "images": {
    "main": "https://example.com/main.jpg",
    "gallery": [
      "https://example.com/photo1.jpg",
      "https://example.com/photo2.jpg"
    ],
    "source": "https://example.com/meeting-room",
    "verified": true,
    "verifiedAt": "2026-03-05T00:00:00Z"
  }
}
```

## 4. 資料清理流程

### 4.1 需要清理的資料類型

| 問題類型 | 說明 | 清理方式 |
|---------|------|---------|
| **重複記錄** | 同一場地有多筆記錄 | 合併或刪除 |
| **錯誤官網** | URL 指向錯誤網站 | 重新搜尋正確官網 |
| **過時資訊** | 價格、電話等已變更 | 從官網更新 |
| **格式不一致** | 電話格式、地址格式 | 標準化格式 |
| **缺失欄位** | 必要欄位為空 | 從官網補充 |
| **錯誤分類** | 場地類型錯誤 | 重新分類 |

### 4.2 清理規則

#### 電話格式

```python
# 標準格式：XX-XXXX-XXXX
正確：02-1234-5678
錯誤：(02)12345678
錯誤：02 1234 5678
```

#### 地址格式

```python
# 標準格式：縣市區段路巷弄號樓室
正確：台北市信義區信義路五段1號
錯誤：台北市信義路五段1號（缺少區）
```

#### 價格格式

```python
# 標準格式：數字（整數）
正確：30000
錯誤："30,000"
錯誤："3萬"
```

#### 容量格式

```python
# 標準格式：數字（整數）
正確：100
錯誤："100人"
錯誤："約100"
```

## 5. 資料確認流程

### 5.1 驗證哪些欄位

| # | 欄位 | 驗證方式 | 通過條件 |
|---|------|---------|---------|
| 1 | `url` | HTTP 請求 | 200 OK，標題包含場地名稱 |
| 2 | `venueListUrl` | HTTP 請求 | 200 OK，頁面有會議室資訊 |
| 3 | `venueMainImageUrl` | HTTP 請求 | 200 OK，圖片可載入 |
| 4 | `contactPhone` | 格式檢查 | 符合 XX-XXXX-XXXX 格式 |
| 5 | `priceHalfDay` | 範圍檢查 | > 0 且 < 1,000,000 |
| 6 | `maxCapacityTheater` | 範圍檢查 | > 0 且 < 10,000 |
| 7 | `roomsCount` | 一致性檢查 | 與實際會議室數量一致 |

### 5.2 驗證流程

#### Phase 1：官網驗證

```python
1. 開啟 url
2. 檢查 HTTP 狀態碼
3. 檢查網頁標題是否包含場地名稱
4. 記錄 verifiedTitle
5. 記錄 lastVerified
```

#### Phase 2：會議室頁面驗證

```python
1. 搜尋會議室相關連結
2. 開啟會議室頁面
3. 記錄 venueListUrl
4. 抓取會議室清單
5. 統計 roomsCount
```

#### Phase 3：照片驗證

```python
1. 檢查 venueMainImageUrl 是否存在
2. 檢查圖片是否可載入
3. 檢查圖片來源是否為官網
4. 記錄 images.verifiedAt
```

#### Phase 4：資料一致性驗證

```python
1. 檢查 roomsCount 與實際數量是否一致
2. 檢查 venueMainImageUrl 與 images.main 是否一致
3. 檢查價格是否合理
4. 檢查容量是否合理
```

### 5.3 驗證失敗處理

| 失敗類型 | 處理方式 |
|---------|---------|
| **官網無法連線** | 搜尋新官網、確認是否歇業 |
| **會議室頁面找不到** | 手動搜尋、標記待修 |
| **照片無法載入** | 重新抓取照片 |
| **資料不一致** | 更新為正確資料 |
| **欄位缺失** | 從官網補充 |

## 7. 常見問題處理

### 7.1 官網無法連線

**可能原因**：
- DNS 解析失敗
- SSL 憑證問題
- 伺服器暫時無法回應
- 網站已關閉

**處理步驟**：
1. 使用 curl 測試：`curl -I https://example.com`
2. 搜尋新官網：`場地名稱 官網`
3. 檢查是否歇業：搜尋 `場地名稱 歇業`
4. 若確認歇業，標記 status = '下架'
5. 若找到新官網，更新 url

### 7.2 找不到會議室頁面

**可能原因**：
- 會議室資訊在首頁
- 會議室資訊在 PDF 中
- 需要登入才能查看
- 使用 JavaScript 動態載入

**處理步驟**：
1. 搜尋所有連結中的「會議」「宴會」關鍵字
2. 檢查是否有 PDF 下載連結
3. 使用 Playwright 載入動態內容
4. 若仍找不到，標記 status = '待修'

### 7.3 照片來源不正確

**可能原因**：
- 使用了 Wikipedia 照片
- 使用了第三方網站照片
- 照片連結失效

**處理步驟**：
1. 重新從官網抓取照片
2. 確認 images.source 為官網 URL
3. 更新 venueMainImageUrl 和 images.main
4. 記錄 images.verifiedAt

### 7.4 國際連鎖酒店被防護系統阻擋

**常見酒店**：
- 萬豪集團（W 飯店、艾美、喜來登）
- 希爾頓
- 洲際（IHG）
- 凱悅

**處理步驟**：
1. 使用 browser 工具手動驗證
2. 記錄會議室頁面 URL
3. 補充到已知正確 URL 清單

### 7.5 場地類型難以判斷

**判斷準則**：

| 場地類型 | 判斷條件 |
|---------|---------|
| **飯店場地** | 提供住宿服務的飯店/酒店 |
| **會議中心** | 專門提供會議服務的場地 |
| **展演場地** | 提供展演空間的場地 |
| **展覽中心** | 大型展覽館 |
| **婚宴場地** | 主要提供婚宴服務 |
| **運動場地** | 體育館、運動中心 |
| **機關場地** | 政府機關、學校場地 |

### 7.6 重複記錄處理

**判斷準則**：
- 同一場地的多個會議室：保留（每個會議室一筆）
- 同一場地重複建立：合併

**處理步驟**：
1. 比對 name 和 address
2. 比對 url
3. 若確定重複，保留資料較完整的一筆
4. 刪除另一筆

4. **逐一檢查**
   - 檢查官網是否可連線
   - 檢查會議室頁面是否正確

   - 檢查資料是否完整
