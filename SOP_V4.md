## 1. 目標與範圍

### 目標
確保台灣會議場地資料庫的**完整性**與**準確性**。

### 核心原則
> **資料對了才有價值，資料錯了一文不值。**
> **一次做對，不要重來。**
> **所有網址都要記錄，下次更新不用重新找。** ⭐ 新增

## 2. 驗證流程

### Phase 1: 官網驗證
```
1. 開啟官網 URL
2. 檢查 HTTP 狀態碼
3. 檢查網頁標題是否匹配
4. 記錄錯誤類型（DNS、SSL、Timeout 等）
```

### Phase 2: 會議室頁面尋找
```
1. 搜尋關鍵字：會議、宴會、場地、租借、活動
2. 找到會議室相關連結
3. 開啟會議室頁面
4. 記錄 meetingPageUrl
```

### Phase 3: 會議室完整清單
```
1. 在會議室頁面尋找所有會議室名稱
2. 記錄每個會議室的：
   - 名稱（roomName）
   - 容納人數（maxCapacity）
   - 價格（如有）
   - 照片（如有）
   - 詳細頁面 URL（如有）
3. 比對資料庫現有記錄
4. 補充缺失的會議室記錄
5. 記錄 venueListUrl（會議室清單頁面）
```

### Phase 4: 照片抓取 ⭐ 更新
```
1. 抓取場地主要照片（venueMainImageUrl）
   - 場地外觀、大廳、建築照
   - 所有會議室共用此照片
2. 抓取每個會議室的照片（roomImageUrl）
   - 會議室內部照
   - 每個會議室可能有不同的照片
3. 記錄照片來源頁面
```

### Phase 5: 狀態判定
```
1. 根據驗證結果判定狀態
2. 更新資料庫欄位
3. 標記問題類型
```

## 3. 網址資訊記錄規範 ⭐ 重要

### 3.1 必須記錄的網址

每個場地必須記錄以下網址資訊：

| 網址類型 | 欄位名稱 | 說明 | 範例 |
|---------|---------|------|------|
| **主網頁** | `url` | 場地的官方網站首頁 | `https://www.ticc.com.tw/` |
| **會議室清單頁** | `venueListUrl` | 會議室清單頁面 | `https://www.ticc.com.tw/venue` |
| **場地主照片** | `venueMainImageUrl` | 場地主要照片（所有會議室共用） | `https://www.ticc.com.tw/images/main.jpg` |
| **會議室照片** | `roomImageUrl` | 會議室照片（每個會議室不同） | `https://www.ticc.com.tw/room/101.jpg` |

### 3.2 網址記錄範例

```json
{
  "id": 1448,
  "name": "台北國際會議中心(TICC)",
  "roomName": "大會堂",
  
  // 必須記錄的網址
  "url": "https://www.ticc.com.tw/",
  "venueListUrl": "https://www.ticc.com.tw/wSite/sp?xdUrl=/wSite/ap/lp_Activity_1.jsp",
  "meetingPageUrl": "https://www.ticc.com.tw/venue/intro",
  
  // 照片
  "venueMainImageUrl": "https://www.ticc.com.tw/var/file/0/1000/main.jpg",
  "images": {
    "main": "https://www.ticc.com.tw/room/大會堂.jpg",
    "gallery": [
      "https://www.ticc.com.tw/room/大會堂-1.jpg",
      "https://www.ticc.com.tw/room/大會堂-2.jpg"
    ],
    "source": "https://www.ticc.com.tw/venue/intro"
  }
}
```

### 3.3 網址記錄原則

| 原則 | 說明 |
|------|------|
| **一次記錄，永久使用** | 記錄所有網址，下次更新不用重新找 |
| **優先使用官方網址** | 避免使用第三方網站的照片 |
| **記錄最深層網址** | 直接連結到會議室頁面，不要只記錄首頁 |
| **照片要記錄來源** | 記錄照片是從哪個頁面抓取的 |

### 3.4 照片分類

| 照片類型 | 欄位 | 說明 |
|---------|------|------|
| **場地主照片** | `venueMainImageUrl` | 場地外觀、大廳、建築照，所有會議室共用 |
| **會議室主照片** | `images.main` | 會議室內部主照片 |
| **會議室相簿** | `images.gallery` | 會議室多張照片 |
| **平面圖** | `images.floorPlan` | 會議室平面圖 |

### 3.5 同一場地多會議室的處理

當一個場地有多個會議室時：

```
場地: 台北國際會議中心(TICC)
├── url: https://www.ticc.com.tw/                    (所有會議室共用)
├── venueListUrl: https://www.ticc.com.tw/venue      (所有會議室共用)
├── venueMainImageUrl: https://xxx/main.jpg          (所有會議室共用)
│
├── 大會堂
│   ├── roomImageUrl: https://xxx/大會堂.jpg         (獨立)
│   └── images.gallery: [...]                        (獨立)
│
├── 101會議室
│   ├── roomImageUrl: https://xxx/101.jpg            (獨立)
│   └── images.gallery: [...]                        (獨立)
│
└── ...
```

## 4. 會議室完整清單

### 4.1 為什麼需要完整清單？

**問題**：早期的驗證只確認官網可開啟，沒有完整抓取所有會議室。
- 政大公企中心：只有 1 筆記錄，實際有 7 個會議室
- TICC：45 筆記錄（正確，每個會議室都有獨立記錄）
- 師大進修推廣學院：13 筆記錄（正確）

**解決方案**：強制要求完整抓取所有會議室，每個會議室都要有獨立記錄。

### 4.2 會議室清單欄位

```json
{
  "venueListUrl": "會議室清單頁面 URL",
  "rooms": [
    {
      "roomName": "會議室名稱",
      "maxCapacityTheater": "劇院式容納",
      "maxCapacityClassroom": "教室式容納",
      "priceHalfDay": "半日價",
      "priceFullDay": "全日價",
      "roomUrl": "會議室詳細頁面 URL",
      "roomImageUrl": "會議室照片 URL",
      "images": {
        "main": "主照片",
        "gallery": ["照片1", "照片2"]
      }
    }
  ],
  "roomsCount": "會議室總數",
  "roomsVerified": "已驗證會議室數"
}
```

## 5. 狀態定義

### status 欄位（後台分類用）

| 狀態 | 值 | 說明 | 後台顯示 |
|------|-----|------|---------|
| **上架** | `上架` | 營運中，會議室清單完整，網址資訊完整 | ✅ 綠色 |
| **下架** | `下架` | 已歇業或停止營運 | ❌ 紅色 |
| **待修** | `待修` | 資料有誤需修正（含會議室不完整、網址缺失） | ⚠️ 黃色 |
| **不提供會議** | `不提供會議` | 非會議場地（如咖啡廳） | 🚫 灰色 |

### verificationStatus 欄位（驗證細節）

| 狀態 | 說明 |
|------|------|
| `已驗證` | 官網、會議室頁面、會議室清單、照片、網址都正確 |
| `待手動驗證` | 被防護系統阻擋（如 Cloudflare） |
| `待確認` | 官網無法連線，需確認營運狀態 |
| `待修正` | 會議室頁面或照片有問題 |
| `會議室不完整` | 會議室清單不完整，需補充 |
| `網址不完整` | 網址資訊不完整，需補充 ⭐ 新增 |

## 7. 問題分類

### 7.1 DNS 無法解析
- **原因**: 官網域名已失效
- **處理**: 
  - 搜尋新官網
  - 確認是否歇業
  - 標記為「待確認」或「下架」

### 7.2 官網標題不匹配
- **原因**: 可能指向錯誤官網或集團首頁
- **處理**: 
  - 手動確認官網正確性
  - 更新為正確的會議室頁面 URL

### 7.3 找不到會議室頁面
- **原因**: 
  - 網站無會議室資訊
  - 純餐飲或展館
  - 需登入才能查看
- **處理**: 
  - 確認是否提供會議服務
  - 標記為「待修」或「不提供會議」

### 7.4 會議室清單不完整
- **原因**: 
  - 只有一筆記錄，實際有多個會議室
  - 早期資料沒有完整抓取
- **處理**: 
  - 重新抓取會議室清單
  - 補充缺失的會議室記錄
  - 標記為「會議室不完整」

### 7.5 網址資訊不完整 ⭐ 新增
- **原因**: 
  - 缺少 venueListUrl
  - 缺少 venueMainImageUrl
  - 缺少會議室照片
- **處理**: 
  - 重新抓取網址資訊
  - 標記為「網址不完整」

### 7.6 缺少會議室照片
- **原因**: 
  - 會議室頁面無照片
  - 照片載入失敗
- **處理**: 
  - 手動補充照片
  - 或標記為「待修」

### 7.7 SSL 憑證錯誤
- **原因**: 憑證過期或設定錯誤
- **處理**: 
  - 嘗試使用 HTTP
  - 通知業者

### 7.8 連線逾時
- **原因**: 網站回應過慢
- **處理**: 
  - 增加逾時時間重試
  - 手動驗證

### 7.9 防護系統阻擋
- **原因**: Cloudflare、Akamai 等
- **處理**: 
  - 標記為「待手動驗證」
  - 這些場地確定有會議室

## 10. 後台分類

### 10.1 後台網址

### 10.2 分類邏輯

```javascript
// 後台分類依據 status 欄位
function getCategory(venue) {
  switch (venue.status) {
    case '上架':
      return 'active';      // 綠色，可租借
    case '下架':
      return 'closed';      // 紅色，已歇業
    case '待修':
      return 'needsFix';    // 黃色，需修正
    case '不提供會議':
      return 'notMeeting';  // 灰色，非會議場地
    default:
      return 'unknown';     // 未分類
  }
}
```

| 下架 | 🔴 紅色 | 已歇業或停止營運 |
| 待修 | 🟡 黃色 | 資料有誤需修正（含會議室不完整、網址缺失） |
| 不提供會議 | ⚪ 灰色 | 非會議場地 |

## 11. 特殊情況處理

### 11.1 國際連鎖酒店（被防護阻擋）
- 萬豪集團（W飯店、艾美酒店）
- 洲際集團（晶華酒店、英迪格酒店）
- 希爾頓集團（寒舍艾麗）
- 凱悅集團（君悅酒店）

**處理方式**:
- 標記為「上架」
- verificationStatus = 「待手動驗證」
- **手動補充會議室清單**
- **手動補充網址資訊** ⭐ 新增
- 手動補充會議室頁面 URL

### 11.2 已歇業場地
- 確認歇業後標記為「下架」
- 記錄歇業時間（如已知）

### 11.3 咖啡廳等非會議場地
- 標記為「不提供會議」
- 從會議場地清單排除

### 11.4 重複場地
- 保留最完整的紀錄
- 刪除重複項

### 11.5 會議室清單不完整
- 重新抓取會議室清單
- 補充缺失的會議室記錄
- 標記為「待修」直到清單完整

### 11.6 網址資訊不完整 ⭐ 新增
- 重新抓取網址資訊
- 補充 venueListUrl
- 補充 venueMainImageUrl
- 補充會議室照片
- 標記為「待修」直到網址完整

## 14. 常見錯誤案例與解決方案

### 14.1 官網 URL 錯誤

**案例：政大公企中心**

| 項目 | 錯誤 | 正確 |
|------|------|------|
| URL | `https://www.nccu.edu.tw` | `https://cpbae.nccu.edu.tw` |
| 問題 | 政大首頁，不是公企中心官網 | 公企中心獨立網站 |
| 結果 | 找不到會議室清單 | 可以找到 7 個會議室 |

**原因分析**：
1. 資料庫中的 URL 是母機構首頁，不是場地官網
2. 子網域 `cpbae.nccu.edu.tw` 是獨立網站
3. SOP 只從首頁找連結，沒有嘗試子網域

**解決方案**：
```
1. 檢查官網 URL 是否為場地專屬網站
2. 如果是母機構首頁，嘗試常見子網域模式：
   - {縮寫}.{學校}.edu.tw（如 cpbae.nccu.edu.tw）
   - {場地名}.{domain}.com.tw
3. 使用搜尋引擎驗證正確官網
4. 更新資料庫中的 URL
```

### 14.2 動態網頁載入問題

**案例：政大公企中心**

| 項目 | 說明 |
|------|------|
| 網址 | `https://cpbae.nccu.edu.tw/cpbae/space/search` |
| 問題 | 頁面使用 JavaScript 渲染，需要等待載入 |
| 解決 | 使用 `waitUntil: 'networkidle'` + `waitForTimeout(5000)` |

### 14.3 子網域命名模式

**常見的子網域模式**：

| 母機構 | 子網域模式 | 範例 |
|--------|-----------|------|
| 政大 | cpbae, sce, pbt | cpbae.nccu.edu.tw |
| 台大 | ntucc, sce | ntuacc.org |
| 師大 | sce | sce.ntnu.edu.tw |
| 清大 | arts, gen | arts.nthu.edu.tw |
| 交大 | nycu | nycu.edu.tw |

### 14.4 檢查清單

當找不到會議室清單時：

- [ ] 確認官網 URL 是場地專屬網站（不是母機構首頁）
- [ ] 嘗試常見的子網域模式
- [ ] 使用 `networkidle` 等待動態內容
- [ ] 搜尋引擎驗證正確官網
