## 一、產品定位

### 存在理由
> **改變世界，讓後 AI 世代，只要有專長就可以有服務機會。**

### 核心價值主張
> **活動大小事，就問活動大師。**

### 產品本質
不是「場地搜尋引擎」，而是：
> **活動生態系 —— 連結需求方、供給方、賦能訓練，讓每場活動都能感動參加者。**

### 北極星指標
> **活動參加者被感動**

---

## 二、目標用戶

| 角色 | 描述 |
|------|------|
| **活動需求方** | 辦活動的人（企業、組織、個人） |
| **活動參加者** | 可轉化為舉辦者 |
| **服務提供者** | 場地、布置、攝影、樂隊、主持人、執行等 |
| **潛在服務者** | 有特長但還沒接案的人 → 提供訓練 |

### 目標活動類型
- 企業員工大會
- 產品發表會
- 婚禮
- 家長會
- 培訓課程
- **所有需要場地的活動**

### 用戶痛點
- 沒時間、不想花太多時間
- 沒經驗、怕踩坑
- 現有方案：Google 搜尋 → 一個個找 → 打電話確認 → 費時

---

## 三、服務範圍

### 場地層
- 場地搜尋（100人以上，藍海）
- 場地資訊（價格、容納人數、照片、設備）
- **媒合 + 預訂**

### AI 顧問層
- **LINE Bot 7/24 服務**
- 深度需求挖掘（不是只回答問題）
- 個人化推薦
- 幫忙 mail 預約

### 服務層
- 布置
- 攝影
- 樂隊
- 主持人
- 活動執行
- 活動橋段設計規劃
- 演講準備協助
- 貴賓聯繫協助

### 人機協作
- AI：預約、初步溝通、推薦
- 人：**議價、簽約、最終確認**

### 訓練賦能層
- 服務提供者培訓
- 認證系統
- 讓有專長的人都能找到服務機會

## 四、市場定位

### 藍海區隔

| | 現有競爭者 | 活動大師 |
|---|-----------|----------|
| 場地規模 | 100人以下 | **100人以上** |
| 服務整合 | ❌ 只有場地 | **場地 + 服務 + 訓練** |
| 服務模式 | 自助搜尋 | **AI 顧問式** |
| 服務時間 | 上班時間 | **7/24 LINE Bot** |
| 需求挖掘 | ❌ | **✅ 深挖需求** |

### 護城河
- 公關公司多年經驗累積
- 市場太小，巨頭不想做
- AI + 人力混合服務
- 完整生態系（需求方 + 供給方 + 訓練）

## 五、商業模式

### 收費模式

| 服務類型 | 收費方式 |
|---------|----------|
| 只租場地 | **免費** + 場地方抽成 |
| 活動規劃 | 整體服務費（含在報價中） |
| 整場活動 | 場地費外 **15% 服務費** |

### 價值交換
- 省 3 小時 = 願意付 500 元
- 省 3 小時以上 = 願意付更多
- 場地方：省成本 + 有顧客 = 願意配合

## 六、技術架構

### 資料結構（兩層設計）

```
場地 (venues)
├── id
├── name（場地名稱）
├── type（飯店/會館/展覽館）
├── address
├── phone
├── email
├── website
├── status（上架/下架/待修）
└── ...

會議室 (rooms)
├── id
├── venue_id → venues.id
├── name（會議室名稱）
├── capacity（容納人數）
├── price_half_day
├── price_full_day
├── equipment
├── images
└── ...

服務商 (service_providers)
├── id
├── name
├── type（布置/攝影/主持人...）
├── portfolio
├── rating
├── certified（是否認證）
└── ...

活動 (events)
├── id
├── user_id
├── venue_id
├── service_ids []
├── requirements（需求記錄）
├── status
└── satisfaction（感動評價）
```

### AI 顧問
- LINE Bot 介面
- LLM 對話
- 需求挖掘邏輯
- 推薦引擎
