## ✅ V4.6 改善措施

### Phase 2: 會議室頁面尋找（強制執行）

```
1. 搜尋關鍵字：會議、宴會、場地、租借、活動
2. 找到會議室相關連結
3. 開啟會議室頁面
4. ⭐ 記錄 meetingPageUrl
```

**改善效果**：
- ✅ 強制找到正確的會議室頁面
- ✅ 記錄 meetingPageUrl（會議室頁面 URL）

---

### Phase 3: 會議室完整清單（強制執行）⭐

```
1. 在會議室頁面尋找所有會議室名稱
2. 記錄每個會議室的完整資訊
3. 統計會議室總數（roomsCount）
4. ⭐ 記錄 venueListUrl（會議室清單頁面）

失敗條件：
- 會議室數量與官網不一致 → 待修
- roomsCount 未記錄 → 待修
- 缺少會議室記錄 → 待修
```

**改善效果**：
- ✅ 強制記錄 venueListUrl
- ✅ 確保 URL 指向會議室清單頁面
- ✅ 驗證 URL 正確性

---

### 第 3 節：網址資訊記錄規範（明確定義）

每個場地**必須記錄**以下網址：

| 網址類型 | 欄位名稱 | 說明 |
|---------|---------|------|
| **主網頁** | `url` | 場地的官方網站首頁 |
| **會議室清單頁** | `venueListUrl` | 會議室清單頁面 ⭐ |
| **場地主照片** | `venueMainImageUrl` | 場地主要照片 |
| **會議室照片** | `roomImageUrl` | 會議室照片 |

**改善效果**：
- ✅ 明確定義 venueListUrl 的標準
- ✅ 強制記錄所有網址
- ✅ 下次更新不用重新找
