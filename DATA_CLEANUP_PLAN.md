## 🎯 清理流程

### 第一階段：緊急修正（預計 1-2 天）

#### 1.1 修正狀態不一致
```bash
# 執行腳本：將缺重要資訊的場地改為下架
node scripts/fix-inconsistent-status.js
```

**處理邏輯**：
- 缺電話 OR 缺地址 OR 缺照片 → 改為「下架」
- 標註原因：`"下架原因": "資料不完整"`
- 記錄修正時間：`"lastFixedAt": "2026-03-07"`

#### 1.2 補充必填資訊
```bash
# 執行腳本：批次補充電話、地址、類型
node scripts/fix-missing-fields.js
```

**處理方式**：
- 從官網或公開資料補充
- 使用爬蟲自動化收集
- 人工驗證關鍵場地

### 第二階段：批次清理（預計 3-5 天）

#### 2.1 照片批次處理
```bash
# 執行腳本：批次爬取場地照片
node scripts/batch-crawl-photos.js
```

**處理方式**：
- 從官網爬取會議室照片
- 篩選高品質照片
- 驗證照片有效性

#### 2.2 價格補充
```bash
# 執行腳本：補充價格資訊
node scripts/fix-missing-prices.js
```

**處理方式**：
- 從官網獲取價格
- 如無公開價格，標註「來電詢問」
- 驗證價格合理性

### 第三階段：驗證與優化（預計 2-3 天）

#### 3.1 資料驗證
```bash
# 執行腳本：驗證所有場地資料
node scripts/validate-all-venues.js
```

**驗證項目**：
- ✅ 電話格式正確
- ✅ 地址真實存在
- ✅ 照片連結有效
- ✅ 價格合理
- ✅ 狀態與資料完整性一致

#### 3.2 資料優化
- 合併重複場地
- 統一命名格式
- 補充遺漏欄位
